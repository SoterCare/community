# Contributing to the SoterCare Community

Thank you for wanting to contribute! This repository holds our community's events, guides, templates, and resources — so you don't need to be a programmer to contribute here. Writers, designers, organizers, and photographers are just as valuable as developers.

## 🌱 First time contributing to open source?

Start with our [First Contribution Guide](guides/first-contribution.md) — it walks you through everything from forking to your first merged pull request.

## Ways to Contribute

| Contribution | Examples |
| --- | --- |
| 📝 Documentation | Fix typos, clarify guides, translate content |
| 🎪 Event content | Add event reports, workshop notes, photos, slides |
| 📚 Resources | Add learning resources, tutorials, tools to `resources/` |
| 🎨 Design | Improve templates, create graphics for `assets/` |
| 💡 Ideas | Open a Community Suggestion or Event Proposal issue |
| 🔍 Review | Review open pull requests and give constructive feedback |

## Workflow

1. **Find or open an issue** — check [open issues](https://github.com/SoterCare/community/issues) or open a new one so we can discuss before you invest time
2. **Fork** the repository and create a branch: `git checkout -b docs/fix-workshop-guide`
3. **Make your changes** — follow the style of the file you're editing
4. **Commit** with a clear message: `docs: clarify speaker checklist timing`
5. **Open a Pull Request** — fill in the PR template; link the issue it closes

## Standards

- **Markdown:** use `##` headings, sentence-case titles, and relative links within the repo
- **File naming:** kebab-case (`event-report.md`), dates as `YYYY-MM-DD` prefixes for event files
- **Images:** compressed (WebP/optimized PNG), stored in `photos/` or `assets/`
- **One PR, one purpose:** keep pull requests focused and small
- **Reports:** use the matching template from [`templates/`](templates/)

## Review Process

A maintainer will review your PR within a few days. We may suggest changes — that's a normal, healthy part of open source, not criticism. Once approved, we merge it and your contribution is recognized in our [Contributor Recognition](docs/recognition.md) system.

## Questions?

Open a [Discussion](https://github.com/SoterCare/community/discussions) or ask in the issue you're working on. Please read our [Code of Conduct](CODE_OF_CONDUCT.md) — it applies to all community spaces.

Thank you for helping build SoterCare ❤️
