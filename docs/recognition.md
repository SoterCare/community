# 🏅 Contributor Recognition

Every contribution to SoterCare gets recognized — code, docs, design, organizing, mentoring, volunteering, and speaking all count equally. Recognition is public, specific, and never pay-to-play.

## How recognition works

| Contribution | Recognition |
| --- | --- |
| First merged PR | Welcome shout-out in the PR + `first-contribution` mention in release notes |
| Any merged PR | Listed in [contributors graph](https://github.com/SoterCare/community/graphs/contributors); credited in reports your content appears in |
| Event volunteering | Named in the event report's credits |
| Speaking | Named in the event report + slides credited in `slides/` |
| Mentoring | Listed in the mentor pool + named by mentees in their milestone posts |
| Organizing | Named as organizer in the report + eligible for maintainer track |

## Levels

Recognition compounds into the role ladder defined in [Governance](../GOVERNANCE.md):

```
Member → Contributor → Volunteer/Mentor → Organizer → Maintainer → Core Team
```

Moving up is always based on **sustained, visible contribution** — never on who you know.

## Hall of Fame

Each semester, the core team highlights standout contributors in a pinned Discussion and on our social channels:

- 🌟 **Contributor of the Semester** — most impactful overall contribution
- 🌱 **Newcomer of the Semester** — best first-semester contributor
- 🧑‍🏫 **Mentor of the Semester** — chosen from mentee feedback
- 🎪 **Event of the Semester** — best-run event, credited to its whole team

## Nominating someone

See someone doing great work? Open a [Community Suggestion](https://github.com/SoterCare/community/issues/new/choose) or post in Discussions — peer recognition means the most.
