# 🏷️ Issue Labels

The labels used in this repository and what they mean. Maintainers apply labels when triaging; contributors filter by them to find work.

## Type

| Label | Meaning |
| --- | --- |
| `bug` | Something is broken (links, info, rendering) |
| `documentation` | Docs, guides, reports, resources |
| `event` | Event-related work or proposals |
| `workshop` | Workshop-related work or proposals |
| `hackathon` | Hackathon-related work or proposals |
| `community` | Community programs, suggestions, applications |
| `proposal` | Needs a decision before work starts |
| `design` | Graphics, templates, visual assets |

## Contributor guidance

| Label | Meaning |
| --- | --- |
| `good first issue` | Reserved for newcomers — scoped, documented, mentor available |
| `help wanted` | Open to any contributor, maintainers won't get to it soon |

## Status

| Label | Meaning |
| --- | --- |
| `status: needs triage` | New, not yet reviewed by a maintainer |
| `status: accepted` | Ready to be worked on |
| `status: blocked` | Waiting on something external |

## Priority

| Label | Meaning |
| --- | --- |
| `priority: high` | Time-sensitive (usually event-driven) |
| `priority: low` | Nice to have, no urgency |

**Conventions:** every new issue gets a type label at triage; `good first issue` implies a maintainer has written enough context for a newcomer to succeed alone.
